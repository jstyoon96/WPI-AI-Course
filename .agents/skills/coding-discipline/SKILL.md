# Coding Discipline

## Core Rule
Optimize for clear, reviewable course materials.

## Ground Before Coding
- Inspect current structure before changing it.
- Read the relevant docs and lab README.
- Check public/private boundaries before staging files.

## Surgical Changes
- Keep edits scoped to the requested lab, doc, or validation rule.
- Do not move instructor-only material into public lab folders.
- Do not move raw datasets into public lab folders or root `data/`.
- Do not remove TODO scaffolding from student notebooks unless replacing it with
  a better student exercise.

## Final Report
Include changed files, checks run, check results, risks, and the next useful
step.
